A **falsifiable-leaf Merkle ontology** for AI governance concepts.

---

## What This Is

A structured classification of AI governance claims where every leaf node:
- States a **falsifiable claim** (not a value judgment, a testable assertion)
- Carries an explicit **falsifier** (the observation that would delete it)
- Has a **status** (PROPOSED / ESTABLISHED / CONTESTED)
- Is **content-addressed** — its hash is derived from its own fields + its children (Merkle)

Facets are **orthogonal siblings**, never ranked against each other. The tree is not a stack or a priority list.

---

## Schema

| Field | Values |
|-------|--------|
| Types | `CATEGORY` `DOMAIN` `FACET` `ENTRY` |
| Status | `AXIOM` `ESTABLISHED` `PROPOSED` `CONTESTED` `SUPERSEDED` |

**Integrity Rules (verified at every build):**
1. `ENTRY` is always a leaf — no children
2. Non-AXIOM `ENTRY` must have `claim` **and** `falsifier`
3. No `FACET` nested under another `FACET` — orthogonal siblings
4. `delete(leaf)` → tree still valid ∧ root hash changes
5. `edit(any field)` → root hash changes (tamper-evident)

## Usage

### Interactive Viewer
Open `index.html` in any browser. No server required.

**Features:**
- Click any node to inspect its claim, falsifier, evidence
- Search across all fields (claims, falsifiers, evidence)
- Filter by status (PROPOSED / ESTABLISHED / CONTESTED)
- Facet quick-nav bar — jump to any facet instantly
- **Prune** any leaf — observe live Merkle re-hash
- **Add Entry** modal — contribute a new falsifiable claim
- Export JSON or copy Markdown

### Python Kernel

```bash
# Build, validate, and export ontology.json + ontology.md
python kernel.py --markdown

# Validate only (exit 0=pass, 1=fail)
python kernel.py --validate-only

# Print statistics
python kernel.py --stats
```

## Content Addressing

```
node_hash = SHA-256(own_fields_json | child_hash_1 | … | child_hash_n)
```

This is a Merkle tree. The root hash is a cryptographic digest of the entire ontology. Any change to any claim, evidence, or falsifier anywhere propagates to the root. Pruning any leaf changes the root but leaves all siblings intact.

---

## Design Principles

1. **Falsifiability over assertion** — Every leaf states the observation that would delete it. Unfalsifiable claims cannot be entered.
2. **Orthogonal facets** — Facets are not ranked. Privacy is not "above" accountability or vice versa.
3. **Content addressing** — The tree is tamper-evident. You cannot change a claim without changing the root.
4. **Delete-without-loss** — Removing a contested entry does not break the tree. The tree is always structurally valid.
5. **Status transparency** — PROPOSED means "author's claim." ESTABLISHED means "externally verifiable." CONTESTED means "the literature genuinely disagrees."

## Contributing a Claim

To propose a new entry:
1. Open `index.html` → click **"+ add entry"**
2. Select the parent facet, provide ID (must start with `E.`), label, status, claim, and falsifier
3. Export the modified JSON
4. Open a PR with the updated `kernel.py` entry and regenerated `ontology.json`

Every PR that adds an entry must include a falsifier. PRs without falsifiers are not merged.
